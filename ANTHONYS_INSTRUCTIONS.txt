Designed by Anthony — single Next.js app (root) — operator notes
================================================================

Location: /Users/anthonyjones/Web Design/Designed_By_Anthony

Layout
------
• One Next.js 16 app at the repo root: `src/` (marketing + `/lighthouse`), `public/`, `next.config.ts`, `vercel.json`.
• Supporting folders at root (not separate deploys from this repo): `customer-site-embeds/`, `vertaflow-crm/` stubs or assets as applicable — see README.

Global theme (single source of truth)
-------------------------------------
• Author global tokens in: `src/styles/theme.css`
• Keep in sync: `src/design-system/tokens.css` (same :root variables/values).
• App CSS entry: `src/design-system/dba-global.css`

First-time setup
----------------
1. cd to Designed_By_Anthony
2. npm install
3. Copy `.env.example` → `.env` / `.env.local` as needed for local dev.

Development
-----------
From repo root:
  npm run dev     — builds `public/scripts/site.js`, then Next on :3000
  npm run build   — prebuild (site script + sync headers) + production build

Cursor / IDE
------------
Open the folder "Designed_By_Anthony" as the workspace root.

Vercel
------
• Root Directory: repository root (`.`).
• Install / build: `npm ci` and `npm run build` (see `vercel.json`).

Cloudflare / GCP / Clerk
------------------------
Follow your platform blueprint; env vars are scoped per Vercel project.

Neon / Agency OS (separate product)
-----------------------------------
• This repo’s public site may POST leads to Agency OS (`PUBLIC_CRM_LEAD_URL`, etc.). Database migrations and CRM app live in their own repos/deployments — not in this Next app’s `package.json` scripts unless you add them back.

Public leads (CRM as source of truth)
--------------------------------------
• Marketing audit/contact flows use env from `src/lib/env/marketing.ts` and related routes under `src/app/api/`.
• Lead payload contract: `src/lib/lead-form-contract.ts`
