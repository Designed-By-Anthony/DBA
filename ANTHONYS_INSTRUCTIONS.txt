Designed by Anthony — Turborepo monorepo (apps/web + apps/api) — operator notes
================================================================================

Location: /Users/anthonyjones/Web Design/Designed_By_Anthony

Layout
------
• Turborepo monorepo at the repo root (Bun workspaces).
• apps/web  — Next.js 16 app (marketing + /lighthouse), deployed on Cloudflare Pages: src/, public/, next.config.ts, wrangler.json
• apps/api  — ElysiaJS Cloudflare Worker (api.designedbyanthony.com): src/routes/, wrangler.json
• packages/shared (@dba/shared) — shared business logic: lighthouse/lib/, lib/
• Supporting folders at root (not separate deploys): customer-site-embeds/, vertaflow-crm/

Global theme (single source of truth)
-------------------------------------
• Author global tokens in: apps/web/src/styles/theme.css
• Keep in sync: apps/web/src/design-system/tokens.css (same :root variables/values).
• App CSS entry: apps/web/src/design-system/dba-global.css

First-time setup
----------------
1. cd to Designed_By_Anthony
2. bun install
3. Copy `.env.example` → `.env` / `.env.local` as needed for local dev.

Development
-----------
From repo root:
  bun run dev        — turbo: starts both Next.js (:3000) and Wrangler API (:8787) in parallel
  bun run dev:web    — Next.js only on :3000
  bun run dev:api    — Wrangler ElysiaJS API only on :8787
  bun run build      — turbo: full production build of both apps
  bun run typecheck  — turbo: tsc --noEmit across all packages

Cloudflare Pages + Workers
--------------------------
• Production deploys from GitHub CI only; local deploy commands are for CI/emergency operator use.
• Cloudflare Pages project root is the repository root (`.`). Root `wrangler.json` is canonical for Pages and points at `apps/web/.vercel/output/static` with `nodejs_compat`.
• Next.js Pages:    bun run deploy:web   (OpenNext build + Pages bundle + wrangler pages deploy from apps/web)
• Next.js Worker fallback: bun run --cwd apps/web deploy:worker
• API Worker:       bun run deploy:api   (wrangler deploy from apps/api)
• Secrets/vars (web): Cloudflare Pages project settings
• Secrets (api):    wrangler secret put <KEY> --cwd apps/api

VertaFlow CRM (separate product)
--------------------------------
• Admin and accounts subdomains redirect to **vertaflow.io** from `apps/web/src/middleware.ts`. This marketing app does not ship the CRM codebase.

Public leads
------------
• Marketing forms POST to **`/api/lead-email`** (routed to the Elysia Worker at api.designedbyanthony.com), which forwards JSON to **`LEAD_WEBHOOK_URL`** (e.g. Convex → Slack). Env: apps/web/src/lib/env/marketing.ts, routes in apps/api/src/routes/.
• Lead payload contract: packages/shared/src/lib/lead-form-contract.ts
